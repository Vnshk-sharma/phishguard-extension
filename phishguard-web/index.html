How It Works

PhishGuard uses rule-based machine learning to analyze URL structure in real time.

1
URL Parsing
The URL is broken into components — hostname, path, query, TLD, subdomains — for individual analysis.
2
Feature Extraction
15+ signals are extracted: URL length, @ symbol presence, IP address usage, HTTPS status, keyword detection, and more.
3
Risk Scoring
Each feature is weighted by its phishing correlation. Scores are summed into a 0–100 risk index.
4
Verdict + Explanation
URLs scoring ≥40 are flagged as phishing. You get a confidence score and plain-English reason for every decision.

What Gets Detected

PhishGuard checks for the most common phishing URL patterns used in real attacks.

🎣
Suspicious TLDs
.tk, .ml, .ga, .cf, .gq, .xyz — free domains heavily abused by phishers.
🔢
IP Address Hosts
Legitimate sites use domain names, not raw IPs like 192.168.1.1.
@
@ Symbol in URL
The @ symbol makes browsers ignore everything before it, hiding the real destination.
🔑
Deceptive Keywords
Words like "login", "verify", "secure", "account" in suspicious contexts.
📏
URL Length
Phishing URLs are often unusually long to obscure the real domain name.
🔀
Redirect Patterns
Parameters like ?url=http or /redirect/ indicate the site forwards to another destination.
🔒
HTTPS Check
HTTP-only sites lack encryption — a common trait in low-effort phishing pages.
🌐
Subdomain Depth
Excessive subdomains like paypal.secure.login.evil.com mimic legitimate sites.
